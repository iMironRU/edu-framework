# D01 · Модель данных

## Сущности домена

### НормативныйПравовойАкт (НПА)

Единица реестра — один нормативный акт федерального уровня.

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный идентификатор, напр. `FZ-273`, `PP-1441` |
| `тип` | enum | Вид акта (см. перечень ниже) |
| `издатель` | string | Минпросвещения, Минобрнауки, Правительство РФ и др. |
| `номер` | string | Официальный номер акта |
| `дата_принятия` | date | Дата подписания / утверждения |
| `дата_вступления` | date | Дата вступления в силу |
| `наименование` | string | Полное официальное название |
| `статус` | enum | `действует` / `утратил_силу` / `изменён` |
| `дата_утраты_силы` | date? | Заполняется если статус = `утратил_силу` |
| `заменён_актом` | string? | ID заменяющего акта |
| `домен_первичный` | DomainID | Основной домен-получатель требований |
| `домены_смежные` | list[DomainID] | Затрагиваемые домены |
| `url_pravo_gov` | url | Ссылка на pravo.gov.ru (первоисточник) |
| `примечание` | string? | Особенности применения |

### Типы НПА (enum)

| Значение | Расшифровка |
|---|---|
| `ФЗ` | Федеральный закон |
| `ПП_РФ` | Постановление Правительства РФ |
| `УП_РФ` | Указ Президента РФ |
| `Распоряжение_Правительства` | Распоряжение Правительства РФ |
| `Приказ_Минпросвещения` | Приказ Министерства просвещения РФ |
| `Приказ_Минобрнауки` | Приказ Министерства науки и высшего образования РФ |
| `Приказ_Минтруда` | Приказ Министерства труда и социальной защиты РФ |
| `Приказ_Рособрнадзора` | Приказ Федеральной службы по надзору в образовании |
| `Постановление_ГГСВ` | Постановление Главного государственного санитарного врача |
| `Письмо` | Письмо / разъяснение ведомства (не НПА, информационный характер) |
| `ГОСТ` | Государственный стандарт |

### ЛокальныйНормативныйАкт (ЛНА)

ЛНА — внутренний документ ОО, принятый во исполнение федерального НПА.

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Внутренний идентификатор ОО |
| `наименование` | string | Название ЛНА |
| `основание` | list[НПА.id] | Федеральные НПА, которым соответствует |
| `дата_принятия` | date | Дата приказа об утверждении |
| `дата_следующего_ревью` | date | Плановая дата актуализации |
| `ответственный` | string | Должность ответственного за актуализацию |
| `статус` | enum | `актуален` / `требует_обновления` / `устарел` |

## ERD (концептуальная схема)

```
НПА  1──────────────────────* ЛНА
     (основание для)

НПА  *──────────────────────* Домен
     (домен_первичный / домены_смежные)

НПА  *──────────────────────1 НПА
     (заменён_актом)
```

## Примечания к реализации

- Реестр в текстовом виде: [`../regulatory-registry.md`](../regulatory-registry.md)
- При реализации в ИС: поле `url_pravo_gov` должно вести на стабильный
  permalink документа, а не на страницу поиска
- Письма и разъяснения включаются в реестр с типом `Письмо` и пометкой,
  что они не являются НПА и носят информационный характер
