# D05-P02: Зачисление обучающегося

**Домен:** D05 · **Применимость:** Все ОО · **Статус:** draft

## Назначение

Официально оформить статус обучающегося, обеспечив возникновение образовательных
отношений между ОО и обучающимся и предоставив ему доступ к образовательным услугам.
Результат — приказ о зачислении, личное дело, договор (при платном обучении).

## Нормативное основание

- **ФЗ-273, ст. 53** — образовательные отношения возникают на основании приказа о зачислении
- **ФЗ-273, ст. 54** — обязательность договора при платном обучении
- **ФЗ-152, ст. 6, ч. 1, п. 1; ст. 9, ч. 1** — обработка персональных данных осуществляется с согласия субъекта

## Входы

- Решение приёмной комиссии / результат конкурса `[ВУЗ, СПО]`
- Заявление обучающегося (или родителя для несовершеннолетних)
- Документ об образовании предыдущего уровня
- Паспорт / документ, удостоверяющий личность
- СНИЛС
- Медицинская справка (если требуется для специальности/направления)

## Шаги

1. Проверка комплектности документов
2. Получение согласия на обработку персональных данных [ФЗ-152, ст. 6, ч. 1, п. 1; ст. 9, ч. 1]
3. Заключение договора об образовании при платном обучении [ФЗ-273, ст. 54]
4. Формирование проекта приказа о зачислении
5. Подписание приказа уполномоченным лицом
6. Регистрация приказа в журнале регистрации
7. Заведение личного дела обучающегося
8. Внесение данных в ИС и ФГИС «Контингент»
9. Выдача студенческого билета и зачётной книжки `[ВУЗ, СПО]`

## Выходы / артефакты

- Приказ о зачислении (OrderType: ENROLL)
- Личное дело обучающегося
- Договор об образовании (при платном обучении)
- Согласие на обработку персональных данных
- Запись в ФГИС «Контингент»

## Участники

| Роль | Действие |
|---|---|
| Инспектор / секретарь приёмной комиссии | Проверяет документы, формирует личное дело |
| Юрист | Проверяет договор об образовании |
| Руководитель ОО / уполномоченный | Подписывает приказ |
| Делопроизводитель | Регистрирует приказ |
| Оператор ИС | Вносит данные в систему |

## Типовые ошибки

- Зачисление без оригинала документа об образовании — нарушение правил приёма
- Отсутствие согласия на ПДн до внесения данных в ИС
- Несоответствие даты приказа дате начала обучения
- Нарушение сроков передачи данных в ФГИС «Контингент»
- Отсутствие медсправки для медицинских / педагогических специальностей

## Лучшие практики

- Чек-лист документов на приёме снижает количество ошибок до нуля
- Автоматическая передача в ФГИС «Контингент» через API при создании приказа
- Электронное согласие на ПДн через ЕСИА исключает бумажный документооборот
