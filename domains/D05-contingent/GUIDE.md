# D05 · Руководство по работе с доменом

## Как определить, что изменение относится к D05

Любое действие, которое влечёт **издание приказа по контингенту**, — это D05.
Если приказа нет, но меняется учебный процесс (расписание, оценки) — это D04.

| Ситуация | Домен |
|---|---|
| Студент подаёт заявление о переводе | D05 |
| Преподаватель выставляет оценки | D04 |
| Студент уходит в академотпуск | D05 |
| Студент сдаёт государственный экзамен | D04 |
| ОО передаёт сведения о выпускниках в ФРДО | D12 (данные рождаются в D05) |
| Стипендия назначается | D09 (данные о статусе — из D05) |

## Жизненный цикл обучающегося

```
Абитуриент → Зачислен → Активный → [Отпуск / Перевод / ИУП] → Выпускник
                ↑                                                     ↓
           Восстановлен ←──── Отчислен с правом восстановления     Диплом
```

## Структура приказа по контингенту

Каждый приказ должен содержать:
1. Вид приказа (из классификатора — см. `data-model/entities.md`)
2. Ссылку на нормативное основание (конкретную статью ФЗ-273 или приказа)
3. ФИО, статус и группу обучающегося (до и после изменения)
4. Дату вступления в силу
5. Подпись уполномоченного лица и регистрационный номер

Классификатор видов приказов: [`data-model/entities.md`](data-model/entities.md#классификатор-приказов)

## Типовые ошибки

**Зачисление без оригинала документа об образовании.**
Даже если абитуриент «принесёт завтра» — приказ о зачислении подписывать нельзя.
Исключение: зачисление «условно» с оговоркой в правилах приёма (только ВУЗы).

**Перепутаны статус «отчислен» и «выпускник».**
Выпускник отчисляется в связи с получением образования — это другой вид приказа (EXPELLED_GRADUATE),
другие права (на дубликат диплома) и другая статистика.

**Задержка передачи данных в ФГИС «Контингент».**
Срок — не позднее следующего рабочего дня после подписания приказа.
Нарушение — основание для предписания при проверке Рободрнадзора.

**Согласие на ПДн получено после внесения данных в ИС.**
Нарушение ФЗ-152. Согласие подписывается до любой обработки персональных данных.

**Студент числится активным после окончания срока академотпуска.**
Если студент не вышел из академотпуска — нужен приказ об отчислении,
а не продление статуса «в отпуске» де-факто.

**Разные данные в ИС и в личном деле.**
После каждого приказа данные в ИС и бумажном личном деле должны совпадать.

## Периодичность контрольных действий

| Действие | Периодичность |
|---|---|
| Сверка данных ИС со ФГИС «Контингент» | Ежемесячно |
| Проверка истечения сроков академотпусков | В начале каждого семестра |
| Формирование отчёта о контингенте (ВПО-1, СПО-1 и др.) | По графику Росстата |
| Передача данных в ФИС ФРД о выданных документах | В течение 60 дней после выдачи |
| Архивация личных дел выпускников | Ежегодно (после выпуска) |

## Сроки хранения личных дел

| Категория | Срок хранения |
|---|---|
| Личные дела выпускников | 75 лет |
| Личные дела отчисленных без получения образования | 5 лет |
| Приказы по контингенту | 75 лет |
| Согласия на обработку ПДн | Срок обучения + 3 года |
