# Обзор API

## Что такое Сервис приёма

ФИС ГИА и Приём (Федеральная информационная система) — государственная платформа для проведения приёмных кампаний в российские вузы. API Сервиса приёма позволяет информационной системе вуза взаимодействовать с ФИС автоматически, без ручного ввода через веб-интерфейс.

## Архитектура взаимодействия

```
ИС вуза  ──────────────────────────────────────────→  ФИС (Сервис приёма)
          POST /api/token/new  (заявление, данные)
          GET  /api/token/own/get  (результат)

ФИС      ──────────────────────────────────────────→  ИС вуза
          GET /api/token/despatch/get  (события с ЕПГУ)
```

Взаимодействие происходит по протоколу HTTP в двух режимах.

## Синхронный режим

Используется только для:
- `/api/session/new` — получение сессионного ключа
- `/api/cls/get` — получение классификаторов

Запрос → немедленный ответ.

## Асинхронный режим

Используется для **всех операций с данными** (создание, редактирование, чтение сущностей).

Схема работы: запрос принимается → сервер возвращает `IdJwt` → по `IdJwt` запрашиваем результат позже.

Подробнее: [async-pattern.md](async-pattern.md)

## Ключевые термины

| Термин | Значение |
|---|---|
| **Token / Токен** | Запрос на выполнение операции с сущностью. Не путать с сессионным ключом |
| **IdJwt** | Идентификатор операции, возвращаемый `/api/token/new`. Используется для получения результата |
| **Session-Key** | Ключ сессии, действует 12 часов. Получается через `/api/session/new` |
| **payload_base64** | XML-документ с данными, закодированный в Base64 |
| **signature_base64** | Электронная подпись по ГОСТ 34.10-2012 в формате detached PKCS#7 |
| **EntityType** | Тип сущности в заголовке токена (например, `ApplicationList`) |
| **Action** | Действие над сущностью: `Add`, `Edit`, `GetBy`, `GetDirect`, `GetAll`, `Remove` |
| **Despatch** | Входящие сообщения от ФИС — события инициированные не вузом (с ЕПГУ, из ЛК и т.д.) |
| **ОГРН / КПП** | Идентификаторы организации, используются для авторизации |

## Перечень запросов

| Endpoint | Метод | Режим | Назначение |
|---|---|---|---|
| `/api/session/new` | POST | Синхронный | Получение сессионного ключа |
| `/api/cls/get` | GET | Синхронный | Получение значений классификаторов |
| `/api/token/new` | POST | Асинхронный | Отправка операции с сущностью |
| `/api/token/delay/get` | GET | Синхронный | Оценка времени ожидания результата |
| `/api/token/own/get` | GET | Синхронный | Получение результата операции |
| `/api/token/despatch/get` | GET | Синхронный | Входящие события от ФИС |
| `/api/file/get` | GET | Синхронный | Получение файла |
| `/api/token/certificate/check` | POST | Синхронный | Проверка подписи |
