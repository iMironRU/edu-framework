# Шаг 1. Настройка справочников вуза

> **Когда делать:** один раз перед стартом первой кампании или при изменении структуры программ  
> **Предварительно:** [авторизация](../getting-started/auth.md), [первый запрос](../getting-started/first-request.md)

## Что нужно сделать на этом шаге

```
1. Загрузить справочники организации (DictionaryValueList)  — необязательно, если не используете
2. Внести направления подготовки (OrgDirectionList)
3. Внести образовательные программы (EducationalProgramList)
```

## 1. Загрузить значения собственных справочников

Если в вузе есть дополнительные параметры не покрытые стандартными классификаторами ФИС.

**Сущность:** `DictionaryValueList`  
**Операции:** Add, Edit, GetAll, Remove  
**XSD:** `OwnXsd/DictionaryValueList/`

Сначала запроси типы справочников (что можно создавать):

```
Action: GetAll
EntityType: DictionaryValueList  — но сначала узнай доступные типы через /api/cls/get?ClsType=DictionaryTypeCls
```

## 2. Внести направления подготовки

**Сущность:** `OrgDirectionList`  
**Операции:** Add, Edit, GetAll, Remove  
**XSD:** `OwnXsd/OrgDirectionList/`

Справочник НП — это список направлений подготовки конкретной организации. Привязан к классификатору `DirectionCls` (федеральный перечень УГСН).

> ⚠️ Изменение НП заблокировано пока есть активная кампания текущего года (ошибка 301). Вносить заранее.

## 3. Внести образовательные программы

**Сущность:** `EducationalProgramList`  
**Операции:** Add, Edit, GetAll, Remove  
**XSD:** `OwnXsd/EducationalProgramList/`

ОП привязывается к НП. Используется при создании конкурсных групп.

## Проверка результата

После заполнения справочников убедись что данные записались:

```
Action: GetAll
EntityType: OrgDirectionList
```

Если список непустой и совпадает с ожидаемым — можно переходить к следующему шагу.
